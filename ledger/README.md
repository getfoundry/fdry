# fdry / ledger

Public, append-only ledger of the fdry treasury. Static files only — no backend, no build step. Deployable to GitHub Pages as-is.

## What's in this directory

| File | What it is |
| --- | --- |
| `index.html` | Single-file dashboard. Open in any browser or serve via GitHub Pages. Uses Chart.js via CDN. |
| `latest.json` | Current snapshot: NAV, holdings, recent transactions. Overwritten on every update. |
| `history.json` | Append-only time series of NAV snapshots. Feeds the chart on the dashboard. |

The dashboard has zero build step. Any static host works:

```bash
# local preview
python3 -m http.server 8000 --directory ledger
# → open http://localhost:8000
```

## `latest.json` shape

Current treasury snapshot. The dashboard accepts several key aliases (e.g. `nav_sol` or `navSol`), but prefer this canonical form:

```jsonc
{
  "as_of": "2026-04-20T04:35:00Z",     // ISO-8601 or unix seconds
  "nav_sol": 1234.5678,                 // total NAV in SOL
  "nav_usd": 185432.10,                 // total NAV in USD
  "prev_nav_sol": 1220.1234,            // previous snapshot, for delta %
  "holdings": [
    {
      "symbol": "SOL",
      "mint": "So11111111111111111111111111111111111111112",
      "amount": 900.0,
      "value_sol": 900.0,
      "value_usd": 135000.00
    },
    {
      "symbol": "USDC",
      "mint": "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v",
      "amount": 50432.10,
      "value_sol": 336.21,
      "value_usd": 50432.10
    }
  ],
  "recent_tx": [
    {
      "signature": "5xk...abc",         // base58 tx signature
      "kind": "swap",                   // swap | deposit | withdraw | fee | rebalance
      "token": "SOL",
      "amount_sol": 12.5,
      "timestamp": 1744000000           // unix seconds or ISO-8601
    }
  ]
}
```

## `history.json` shape

Append-only NAV time series. Each entry is one snapshot. Either a bare array or `{ "points": [...] }` is accepted:

```jsonc
[
  { "timestamp": "2026-04-18T00:00:00Z", "nav_sol": 1200.00, "nav_usd": 180000.00 },
  { "timestamp": "2026-04-19T00:00:00Z", "nav_sol": 1220.12, "nav_usd": 183018.00 },
  { "timestamp": "2026-04-20T00:00:00Z", "nav_sol": 1234.57, "nav_usd": 185185.50 }
]
```

Keep entries sorted oldest → newest. The dashboard plots the last 30 points.

## Verifying on-chain

Everything in this ledger is reproducible from Solana mainnet. Cross-check before trusting any number.

- **Treasury account / holdings** — paste a `mint` or treasury address into:
  - Solscan: <https://solscan.io/account/ADDRESS>
  - Solana Explorer: <https://explorer.solana.com/address/ADDRESS>
  - Solana Beach: <https://solanabeach.io/address/ADDRESS>

- **Transactions** — paste a `signature` from `recent_tx`:
  - Solscan: <https://solscan.io/tx/SIGNATURE>
  - Solana Explorer: <https://explorer.solana.com/tx/SIGNATURE>

- **Token prices (for USD conversion)** — Jupiter price API:
  - <https://price.jup.ag/v6/price?ids=SOL>

The dashboard renders Solscan links inline — click any tx signature or mint address to verify.

## Update cadence

`latest.json` and `history.json` are regenerated by the treasury bot. Cadence and writer are documented in `/bot` once that ships. Until then: treat `as_of` on `latest.json` as authoritative.
